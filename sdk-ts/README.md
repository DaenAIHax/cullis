# @agent-trust/sdk

TypeScript SDK for Cullis.

Provides a `BrokerClient` class that handles all agent-to-broker communication,
including x509 + DPoP authentication (RFC 9449), E2E encryption
(AES-256-GCM + RSA-OAEP), message signing (RSA-PSS-SHA256), and session management.

## Requirements

- Node.js >= 18
- Agent x509 certificate + RSA private key (generated by the Cullis PKI)

## Install

```bash
npm install @agent-trust/sdk
```

## Quick Start

### 1. Create a client and authenticate

```typescript
import { BrokerClient } from "@agent-trust/sdk";

const client = new BrokerClient({
  baseUrl: "https://broker.example.com",
  verifyTls: true,
});

await client.login(
  "acme::buyer-agent",    // agentId
  "acme",                 // orgId
  "./certs/buyer.crt",    // certificate path
  "./certs/buyer.key",    // private key path
);
```

### 2. Discover agents

```typescript
const suppliers = await client.discover(["order.read", "order.write"]);
console.log(`Found ${suppliers.length} supplier(s)`);
```

### 3. Open a session and send a message

```typescript
const sessionId = await client.openSession(
  suppliers[0].agent_id,
  suppliers[0].org_id,
  ["order.write"],
);

// Wait for acceptance...
// (poll listSessions until status === "active")

await client.send(sessionId, "acme::buyer-agent", {
  type: "order_request",
  text: "Quote for 1000 M8 bolts please.",
  order_id: "ORD-2026-001",
}, suppliers[0].agent_id);
```

### 4. Poll for messages

```typescript
const messages = await client.poll(sessionId, -1);
for (const msg of messages) {
  console.log(`${msg.sender_agent_id}: ${msg.payload.text}`);
}
```

### 5. RFQ flow

```typescript
// Broadcast a request for quote
const rfq = await client.createRfq(
  ["order.write"],
  { item: "M8 bolts", quantity: 1000 },
  300, // timeout in seconds
);

// Check quotes
const updated = await client.getRfq(rfq.rfq_id);
for (const quote of updated.quotes) {
  console.log(`Quote from ${quote.agent_id}:`, quote.payload);
}
```

### 6. Close session

```typescript
await client.closeSession(sessionId);
```

## Architecture

```
Agent (TypeScript)          Broker                  Agent (Python/TS)
  |                           |                           |
  |-- x509 + DPoP login ---->|                           |
  |<-- access_token ---------|                           |
  |                           |                           |
  |-- open session ---------->|-- notify target -------->|
  |                           |<-- accept session -------|
  |                           |                           |
  |-- E2E encrypt + sign ---->|-- forward (opaque) ----->|
  |                           |<-- E2E encrypt + sign ---|
  |<-- forward (opaque) -----|                           |
  |-- decrypt + verify -------|                           |
```

All messages are E2E encrypted (AES-256-GCM + RSA-OAEP) and signed
(RSA-PSS-SHA256). The broker only sees opaque ciphertext and verifies
the outer transport signature.

## Protocol Compatibility

This SDK implements the exact same wire protocol as the Python SDK
(`agents/sdk.py`). TypeScript and Python agents can communicate
transparently through the same broker.
